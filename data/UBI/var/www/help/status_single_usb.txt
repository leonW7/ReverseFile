:|controllerStatus|:
<p><b>Controller Status </b><br>
<b>WLAN controller Operational Status :</b>This status field
            displays the operational status of the WLAN controller. The WLAN
            controller may be configured as enabled, but is operationally
            disabled due to configuration dependencies. If the operational
            status is disabled, the reason will be displayed in the following
            status field.<br>
            The WLAN controller is composed of multiple components, and each
            component in the system must acknowledge an enable or disable of the
            WLAN controller. During a transition the operational status might temporarily show a pending status.<br>
			<br>
			<b>WLAN controller Disable Reason :</b> If the status is disabled, this field appears and one of the following reasons is listed.<br>
			<b>None:</b> The cause for the disabled status is unknown.<br>	
			<b>Administrator disabled:</b> The Enable WLAN controller option on the global configuration page has been cleared.<br>	
			<b>No IP Address:</b> The WLAN interface does not have an IP address.<br>	
			<b>No SSL Files:</b> The Unified Wireless controller communicates
            with the APs it manages by using Secure Sockets Layer (SSL)
            connections. The first time you power on the Unified Wireless
            controller, it automatically generates a server certificate that
            will be used to set up the SSL connections. The SSL certificate and
            key generation can take up to an hour to complete.<br>
			If routing is enabled on the controller, the operational status might be disabled due to one of the following reasons:<br>	
			<b>No Loopback Interface:</b> The controller does not have a loopback interface.<br>	
			<b>Global Routing Disabled:</b> Even if the routing mode is enabled
            on the WLAN controller interface, it must also be enabled globally
            for the operational status to be enabled.<br>
			<br>	
			<b>IP Address :</b>IP address of the controller.<br>
			<br>
			<b>Peer Controllers :</b> Number of peer WLAN Controllers detected on the network.<br>
			<br>
			<b>Cluster Controller :</b> Indicates whether this controller is the
            Cluster Controller for the cluster.<br>
            Among a group of peer Controllers, one of the Controllers is
            automatically elected or configured to be the Cluster Controller.
            The Cluster Controller gathers status and statistics about all APs
            and clients in the peer group.<br>
            Note: Only the Cluster Controller controller can display managed
            APs, clients, statistics, and RF Scan databases for the whole
            cluster. The Controllers that are not Cluster Controllers can
            display information only about locally attached devices.<br>
			<br>
			<b>Cluster Controller IP Address :</b>The IP address of the peer controller that is the Cluster Controller.<br>
			<br></p>

<p><b>WDS AP Status</b><br><br>
The WDS AP Group Status page displays detailed information about the configured APs and links in the WDS Group. From this page, you can also send a new password to group members.<br><br>
The following fields are on the WDS AP Group Status page.<br><br>
<b>ID:</b>Use the drop-down menu above the fields to select the group number that identifies the configured WDS AP group.<br><br>
<b>Configured AP Count:</b>Number of APs configured in this WDS AP group<br><br>
<b>Connected AP Count:</b>Number of APs managed by the controller that are members of this WDS AP Group. This number is the sum of the Connected Root APs and Connected Satellite APs.<br><br>
<b>source AP Count:</b>Number of Root APs currently being managed by the controller that are members of this WDS AP Group.<br>
<b>Destination AP Count:</b>Number of Satellite APs currently being managed by the controller that are members of this WDS AP Group.<br><br>
<b>Source Bridge AP MAC:</b>MAC Address of the device elected as the Spanning Tree Root Bridge. If spanning tree is disabled this value is 00:00:00:00:00:00.<br><br>
<b>Source Device Type:</b>The type of device elected as the Spanning Tree Root bridge(like None (STP is disabled),Root AP,Satellite AP,External Device (STP Root is not one of the APs))<br><br>
<b>Config WDS Link Count:</b>Number of configured bidirectional links in the WDS AP Group.<br><br>
<b>Detect WDS Links Count:</b>Number of WDS links detected in the system. APs on both sides of the link must detect each other in order for the link to be counted.<br><br>
<b>Blocked WDS Link Count:</b>Number of WDS links blocked by the spanning tree protocol. If the AP on one side of the link reports the link as blocking then the link is counted by this status parameter.<br><br>
<b>WDS Group Password Change Status:</b>Status of the last attempt to configure the password for the WDS Group (like Not Started,Success,Invalid Password,Requested,Timed Out)<br><br>
<b>Edit Password:</b>To change the password for all controlleres and
APs in this WDS Group, select the Edit checkbox, type the new password, and
then click Apply Password. Password must be minumum of 8 characters and can be
up to 63 characters in length.<br><br>
Click <b>Save </b> to save your changes.<br>
				<br>
				Click <b>Cancel </b> to revert to the previous settings.

:|wdsApStatus|:
<p><b>WDS AP Status Summary</b><br><br>
The following fields are on this page.<br><br>
<b>Group ID:</b>Use the drop-down menu above the fields to select the group number that identifies the configured WDS AP group.<br><br>
<b>AP MAC Address:</b>Identifies the AP in the group by its MAC address<br><br>
<b>AP Connection Status:</b>Indicates whether the AP is currently being managed by one of the controlleres in the cluster.<br><br>
<b>Satellite Mode:</b>Indicates whether the AP is a Satellite AP connected to the network via a WDS link or a Root AP connected to the network via a wired link.<br><br>
<b>STP Root Mode:</b>Indicates whether this AP is the root of the spanning tree. If spanning tree is disabled then the AP is always reported as Not STP Root.<br><br>
<b>Root Path Cost:</b>Spanning Tree Path Cost to the root. The root AP always reports this value as 0. If spanning tree is disabled the value is also 0.<br><br>
<b>Ethernet Port STP State:</b>When spanning tree is enabled on the APs in the WDS group this status parameter reports the spanning tree status of the Ethernet port.<br><br>
<b>Ethernet Port Mode:</b>On Satellite APs the Ethernet port can be manually disabled. On root APs the port is always enabled.<br><br>
<b>Ethernet Port Link State:</b>When the Ethernet port is enabled, this status
reports the link state of the port.<br><br></p>

:|wdsLinkStatus|:
<p><b>WDS Link Status</b><br><br>
The WDS Link Status page displays summary information about the link configuration and link state in a WDS Group.<br><br>
The following fields are on the WDS Link Status page.<br><br>
<b>ID:</b>The group number that identifies the configured WDS AP group.<br><br>
<b>Source  AP MAC:</b>The MAC address of one end-point of the WDS link<br><br>
<b>Source AP Radio:</b>The radio number of the WDS link endpoint on the source AP.<br><br>
<b>Destination AP MAC:</b>The MAC address of the Destination AP in the group.<br><br>
<b>Destination AP Radio:</b>The radio number of the WDS link endpoint on the destination AP.<br><br>
<b>Source AP End-Point :</b>Indicates whether the AP specified by the destination MAC detected the AP specified by the source MAC.<br><br>
<b>Destination AP End-Point:</b>Indicates whether the AP specified by the source MAC detected the AP specified by the destination MAC.<br><br>
<b>Aggregation Mode:</b>When parallel links are defined between two APs, this field indicates whether this link is part of the aggregation link pair.<br><br>
<b>Source AP STP:</b>Spanning Tree State of the link on the source AP, which is one of the following:<br><br>Disabled (STP is disabled or Link is down)<br><br>Forwarding<br><br>Learning<br><br>Listening<br><br>Blocking<br><br>
<b>Destination AP STP:</b>Spanning Tree State of the link on the destination
AP, which is one of the following:<br><br>Disabled (STP is disabled or Link is
down)<br><br>Forwarding<br><br>Learning<br><br>Listening<br><br>Blocking<br><br></p>

<p><b>RADIUS Accounting Server Statistics</b><br><br>
<b>Accounting Server Host Address :</b> Use the drop-down menu to select the IP
address of the RADIUS accounting server for which to display statistics.<br>
<br>
<b>Round Trip Time (secs) :</b> Displays the time interval, in hundredths of a second,
between the most recent Accounting-Response and the Accounting-Request that
matched it from this RADIUS accounting server.<br>
<br>
<b>Accounting Requests :</b> The number of RADIUS Accounting-Request packets
sent to this server. This number does not include retransmissions.<br>
<br>
<b>Accounting Retransmissions :</b> The number of RADIUS Accounting-Request
packets retransmitted to this server.<br>
<br>
<b>Accounting Responses :</b> Displays the number of RADIUS packets received on
the accounting port from this server.<br>
<br>
<b>Malformed Access Responses :</b> Displays the number of malformed RADIUS
Accounting-Response packets received from this server. Malformed packets include
packets with an invalid length. Bad authenticators and unknown types are not
included as malformed accounting responses.<br>
<br>
<b>Bad Authenticators :</b> Displays the number of RADIUS Accounting-Response
packets that contained invalid authenticators received from this accounting
server.<br>
<br>
<b>Pending Requests </b> The number of RADIUS Accounting-Request packets
destined for this server that have not yet timed out or received a response.<br>
<br>
<b>Timeouts :</b> The number of accounting timeouts to this server.
Unknown Types - The number of RADIUS packets of unknown type which were received
from this server on the accounting port.<br>
<br>
<b>Packets Dropped :</b> The number of RADIUS packets received from this server
on the accounting port and dropped for some other reason.<br>
</p>
